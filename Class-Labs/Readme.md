## Contributing

We welcome contributions from the community! Whether you're adding a brand-new lab or improving an existing one, please follow the templates below to keep everything consistent and high quality.

---

### Lab Documentation Template

When creating or updating a lab, your `README.md` inside the lab folder must follow this structure:

---

```markdown
# Lab Name

## 1. Overview
<!-- What is this lab about? What are we building? Give a clear 2-4 sentence summary. -->

Example:
> In this lab, we build an AI-powered contract analysis tool using fine-tuned LLMs.
> The goal is to understand how domain-specific training improves model accuracy on legal documents.

---

## 2. Problem Statement
<!-- What real-world problem does this lab solve? Why does it matter? -->

Example:
> Manual contract review is time-consuming and error-prone. This lab automates clause
> identification and risk flagging, reducing review time from hours to minutes.

---

## 3. Prerequisites
<!-- List everything the user needs before starting — tools, accounts, knowledge. -->

- [ ] Tool/Account (e.g., OpenAI API key)
- [ ] Tool/Account (e.g., n8n installed or cloud account)
- [ ] Skill/Knowledge (e.g., Basic understanding of REST APIs)
- [ ] Any files or datasets to download

---

## 4. 🛠️ Steps & Instructions
```

### 🔀 Pull Request Template

When submitting a PR, use the following format for your PR title and description:

---

```
PR Title:  [New Lab / Update] – <Lab Name> – <Section Updated>

Example: [New Lab] – n8n Agentic RAG – Full Lab
Example: [Update] – Fine-Tuning Lab – Section 4 (Steps & Instructions)
```

**PR Description:**

```markdown
## Name
<!-- Your full name or GitHub handle -->

## Description

### What have you done?
<!-- Briefly explain what this PR adds or changes -->

### Type of change
- [ ] Creating a new lab
- [ ] Updating an existing lab

### Which section(s) did you add or update?
- [ ] Section 1 – Overview
- [ ] Section 2 – Problem Statement
- [ ] Section 3 – Prerequisites
- [ ] Section 4 – Steps & Instructions (screenshots included)
- [ ] Section 5 – What You Learned
- [ ] Section 6 – Resources & Links
- [ ] Updated the main Labs Directory table in `Readme.md`

## Screenshots
<!-- If updating Section 4, attach at least one screenshot here to confirm visuals are included -->

## Checklist
- [ ] I followed the Lab Documentation Template
- [ ] Screenshots are included for all major steps (Section 4)
- [ ] Links in the main `Readme.md` table are working
- [ ] No sensitive credentials or API keys are included
```
